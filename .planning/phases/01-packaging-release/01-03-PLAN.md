<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repository/build inputs to frozen bundle | Python modules and approved native binaries are collected into an executable trusted by end users. |
| Onedir output to portable archive | The archive must preserve required files without adding secrets, models, user data, or original media. |
| Archive to clean extraction | Integrity and launch behavior must survive transport and extraction to a new path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-10 | Tampering | Portable ZIP | high | mitigate | Recompute SHA-256 independently and require equality with both SHA256SUMS and BUILD_MANIFEST before accepting the release. |
| T-01-11 | Spoofing | Frozen executable proof | high | mitigate | Execute both the onedir EXE and the independently extracted EXE; retain paths, exit codes, and raw self-test output. |
| T-01-12 | Information Disclosure | Archive contents | high | mitigate | Inspect the archive and reject models, credentials, configuration, job data, transcripts, slide images, and original media. |
| T-01-13 | Denial of Service | Missing frozen dependencies | high | mitigate | Inspect required runtime layout and run real frozen self-tests after packaging and extraction. |
| T-01-14 | Repudiation | Release provenance | medium | mitigate | Persist build output, checksums, manifest data, archive checks, exact executable paths, and self-test results in PACKAGED_VALIDATION.md. |
| T-01-15 | Elevation of Privilege | Unsigned portable executable | low | accept | Code signing is outside this phase; the release remains portable/unsigned and integrity is provided by published SHA-256 evidence. |
</threat_model>
