| Boundary | Description |

| Repository root to cleanup/backup operations | Destructive or moving operations must remain inside exact allowed direct children and must not follow reparse paths. |
| Previous release to staging/publication | A known-good release must survive all build/validation failures and be restorable during a failed swap. |
| Onedir/staged ZIP to published release | Inventory, content exclusions, checksums, and frozen execution determine whether staged artifacts become user-visible. |
